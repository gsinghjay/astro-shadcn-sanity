## astro-app â€” public APIs

All SSG routes are HTML; the APIs below are callable endpoints.

### `GET /api/events/[slug].ics`
iCalendar export of a single event.

- **Auth:** public.
- **Response:** `text/calendar; charset=utf-8`, `Content-Disposition: attachment`.
- **Body:** `VCALENDAR` containing one `VEVENT` generated by `astro-app/src/lib/ical.ts` from the Sanity `event` document.
- **Errors:** `404` if the slug does not match a published event.

### `GET /rss.xml`
Article RSS feed.

- **Auth:** public.
- **Response:** `application/xml`.
- **Implementation:** `@astrojs/rss` using `ALL_ARTICLES_QUERY` (published, current site).

### `GET /robots.txt`
- **Auth:** public.
- **Response:** `text/plain`.
- **Rules:** excludes `/portal/`, `/auth/`, `/student/`, `/demo/`. Mirrors sitemap exclusions in `astro.config.mjs`.

### `POST /api/subscribe`
Newsletter signup.

- **Auth:** public. Rate-limited by middleware.
- **Request:** `application/json` â€” `{ email: string, source?: string }`.
- **Behavior:**
  1. Validate email via zod.
  2. Insert into `subscribers` (Drizzle upsert on unique `email`).
  3. Optionally forward to Resend audience for double opt-in.
- **Response:**
  - `200 { ok: true }` on success.
  - `400` on validation failure.
  - `429` if rate-limited.

### `DELETE /api/subscribe`
Unsubscribe.

- **Auth:** public (token-based link from email).
- **Request:** `?token=...` or `application/json { token }`.
- **Response:** `200 { ok: true }` or `404` if token invalid.

### `ALL /api/auth/[...all]`
Better Auth handler â€” OAuth callback, magic link, session routes.

- **Auth:** provider-managed.
- **Providers:** `google`, `github`, `email` (Magic Link via Resend).
- **Delegated to:** `better-auth`'s Astro integration. All OAuth callback URLs, PKCE flows, and session mutations flow through this mount.
- **Notable:** session cookies are HttpOnly, Secure, SameSite=Lax; signed with `BETTER_AUTH_SECRET`.

## astro-app â€” portal APIs (auth-gated)

All `/portal/api/*` routes require a valid session cookie and a `user.role === 'sponsor'` (except `/portal/api/db-health`, which only requires a session). Middleware denies unauthenticated traffic before the handler runs.

### `GET /portal/api/me`
Return the current user + their resolved sponsor entity.

- **Auth:** session required.
- **Response:** `200 application/json`
  ```ts
  {
    user: { id: string; email: string; name: string; role: 'sponsor' | 'student' };
    sponsor: { _id: string; name: string; slug: string; tier: string; logo: SanityImage } | null;
  }
  ```
- **Errors:** `401` if session invalid, `403` if role denies access.

### `GET /portal/api/projects`
List the authenticated sponsor's projects.

- **Auth:** session + role=sponsor.
- **Query params:** none.
- **Response:** `200 application/json`
  ```ts
  Array<{
    _id: string;
    title: string;
    slug: string;
    status: string;
    semester: string;
    year: number;
    techStack: string[];
    githubRepo: { repoFullName: string; repoUrl: string } | null;
  }>
  ```
  Data is a Drizzle join of Sanity `project` documents where `project.sponsor._ref == currentSponsorId` with rows from `project_github_repos`.

### `GET /portal/api/events`
List the sponsor's events (upcoming + past).

- **Auth:** session + role=sponsor.
- **Response:** `200 application/json` â€” list of event summary objects (title, slug, date, location, status).

### `GET /portal/api/db-health`
D1 heartbeat.

- **Auth:** session required (any role).
- **Response:** `200 { ok: true, dbTime: ISO8601 }` or `500 { ok: false, error }`.

### `GET /portal/api/github/repos`
List the current user's GitHub repositories (via OAuth token).

- **Auth:** session + role=sponsor + GitHub OAuth account linked.
- **Behavior:** reads `account.accessToken` from `account` table for providerId=`github`, calls `GET https://api.github.com/user/repos?per_page=100&sort=updated`.
- **Response:** `200 [{ id, full_name, html_url, description, language, updated_at }]`.
- **Errors:** `403` if no GitHub account linked; `502` if GitHub upstream fails.

### `POST /portal/api/github/links`
Link a GitHub repo to a project.

- **Auth:** session + role=sponsor + project belongs to sponsor.
- **Request:** `{ projectId: string, repoFullName: string, repoUrl: string }`.
- **Behavior:** upsert into `project_github_repos` (unique on `(userId, projectId)`).
- **Response:** `200 { ok: true }`.

### `DELETE /portal/api/github/links`
Unlink.

- **Request:** `{ projectId: string }`.
- **Response:** `200 { ok: true }` or `404` if link did not exist.

## rate-limiter-worker â€” Durable Object RPC

Not an HTTP contract â€” exposed via binding. The Worker itself serves `GET /` returning `"rate-limiter-worker OK"` for health checks.

### `checkLimit(windowMs: number, maxRequests: number): Promise<CheckResult>`

Called from `astro-app/src/middleware.ts` via the DO stub:

```ts
const id = env.RATE_LIMITER.idFromName(clientIp);
const stub = env.RATE_LIMITER.get(id);
const result = await stub.checkLimit(60_000, 100);
```

- **Input:** window size in milliseconds, max requests per window.
- **Output:** `{ allowed: boolean, remaining: number, retryAfterMs: number }`.
- **State:** stored in the DO's SQLite (`ctx.storage.sql`) â€” a rolling log of request timestamps for that DO instance.
- **Alarm-based cleanup:** expired entries trimmed on next request (no separate background sweeper).

## Error conventions

| Code | Meaning |
|---|---|
| 200 | Success |
| 204 | Success, no body |
| 400 | Malformed / failed validation |
| 401 | Missing / invalid session |
| 403 | Authenticated but role doesn't permit |
| 404 | Resource not found |
| 429 | Rate-limited (sliding window) â€” includes `Retry-After` header (seconds) |
| 500 | Unexpected error; usually a Sanity or D1 fault |
| 502 | Upstream provider (GitHub, Resend, Discord) failed |

Responses are JSON except for ICS, RSS, robots.txt, and Better Auth redirects.

## Stability

- Portal APIs are **internal** â€” they are only called from portal pages on the same origin. No external consumer contract is maintained.
- Public APIs (`/api/subscribe`, `/api/auth/**`, `/rss.xml`, `/api/events/*.ics`) are stable; breaking changes require a version bump and CHANGELOG entry.
- `platform-api` v1 namespace is a placeholder until the first consumer ships.

## Security notes

- Every write route must be invoked over HTTPS (CF enforces).
- Portal write routes (`POST/DELETE /portal/api/github/links`) verify the project belongs to the authenticated sponsor server-side; clients cannot cross-link.
- GitHub API tokens are read server-side from `account.accessToken`. They are never forwarded to the browser.
- Sanity read tokens are server-side only (`SANITY_API_READ_TOKEN`) and only used for preview/draft fetches â€” never shipped to clients.
- Rate limiter is the primary DoS guard; pair with Cloudflare's edge DDoS + Turnstile on contact forms.
