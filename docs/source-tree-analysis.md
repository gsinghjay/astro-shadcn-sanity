# Source Tree Analysis

> Annotated source tree for the `astro-shadcn-sanity` monorepo. Reflects the post-Astro-6 / Cloudflare Workers cutover (PR #681, ~Apr 2026) at project version **2.0.0** on branch `preview`.

## Top-level layout

```
astro-shadcn-sanity/
├── astro-app/                  # Workspace: Astro 6.1.10 SSR site (the user-facing product)
├── studio/                     # Workspace: Sanity Studio v5.23 (3 workspaces: capstone, rwc-us, rwc-intl)
├── platform-api/               # Workspace: FastAPI Python Cloudflare Worker (development scaffold)
├── rate-limiter-worker/        # Standalone TS Worker — Durable Object SlidingWindowRateLimiter (production)
├── event-reminders-worker/     # Standalone TS Worker — daily 09:00 UTC cron (production: ywcc-event-reminders)
├── discord-bot/discord-bot/    # Standalone Python FastAPI + discord.py (scaffold; production bots are CF Workers `capstone-bot` + `capstone-ask-worker`)
├── _bmad/                      # BMAD agent skills installation (scripts/, custom/, bmm/, …)
├── _bmad-output/               # BMAD planning artifacts (project-context.md + 314 docs across stories/epics/research)
├── _templates/                 # Cookiecutter-style FastAPI Worker template (`_templates/fastapi-cf-worker/`)
├── _wp-scrape/                 # Legacy WordPress scrape (out-of-scope; do not touch)
├── docs/                       # AI-retrieval doc set (this file lives here) + .archive/ for old scan reports
├── rules/                      # 16 .mdc Sanity agent rule files (referenced from AGENTS.md)
├── scripts/                    # Dev / capture helpers (figma-capture, etc.)
├── tests/                      # Root Playwright E2E specs (20 files × 5 browsers)
├── wiki/                       # 31 long-form human-authored topic guides
├── playwright.config.ts        # E2E config — auto-builds astro-app + serves preview before tests
├── docker-compose.yml          # 5 services: astro-app, studio, astro-rwc-us, astro-rwc-intl, storybook
├── Dockerfile                  # Multi-target Node 24 dev image (dev-astro / dev-studio / dev-storybook)
├── package.json                # Root workspace manifest (v2.0.0); concurrent dev orchestration
├── CLAUDE.md                   # Repo-wide AI agent instructions (project rules + commands)
├── AGENTS.md                   # Sanity + MCP agent setup
├── DESIGN.md                   # NJIT design-system spec (Swiss grid, flat depth)
├── README.md                   # Public README (557 lines)
└── CHANGELOG.md                # semantic-release output (v2.0.0 current; >800 lines)
```

| `sponsors/`     | 2     | prerender                             |  |
| `student/`      | 1     | SSR                                   |  |

## event-reminders-worker/ — Production cron Worker

```
event-reminders-worker/
├── package.json
├── wrangler.toml               # cron: "0 9 * * *" (daily 09:00 UTC); D1 binding to ywcc-capstone-portal
├── src/index.ts                # ~230 LOC — scheduled() handler + Sanity events query + D1 dedupe + Resend + Discord webhook
└── src/__tests__/index.test.ts
```

Deployed Worker name: **ywcc-event-reminders**. Reads upcoming events from Sanity, dedupes against the `sent_reminders` D1 table, sends email via Resend, posts a Discord notification. Shares D1 with `ywcc-capstone` — migrations 0005 / 0006 must stay coordinated.
